    • Prerequisites
    Like any other complex piece of software, wxPython requires other software in order to function properly. Obviously you'll need Python itself, but if you're reading this you've probably already got Python and are just here looking for the best GUI toolkit. Check out the details for your platform of choice here:

    Win32

    • If you have a modern up to date version of Windows and use the binary installer for wxPython found below, you probably don't need anything else.

    • If your tree controls have strange background colors, try loading this MS Common Controls Update as wxWindows does something that causes a bug in one of the older versions to manifest itself. Another way to get this update is to install Internet Explorer or MS Office 200 apps, so if the system has those already then you probably don't need to worry about this.

    • Windows 95 users will probably also need the following items to update their systems:
      • Winsock 2.0
      • OpenGL
      • HTML Help viewer (to view the online documentation)


    Linux/Unix/Etc.

    • The first thing you'll need are the glib and gtk+ libraries. Before you run off and download the sources check your system, you may already have it. Most distributions of Linux come with it and you'll start seeing it on many other systems too now that Sun and others have chosen GNOME as the desktop of choice. If you do have it already then be sure that your installation included the development bits too, not just the shared libraries. For example, on RedHat there are separate RPMs with "-devel" in the name that you'l need to install. If you don't have glib and gtk+ already, you can get the sources here. Build and install them following the directions included.

    • The next item on the list is wxGTK, which is the gtk+ version of wxWindows. This is where the core functionality that wxPython wraps is located. You can get RPMs and source tarballs here, and I have mirrored them here. You'll want to get a version that matches the version number of wxPython you are using. See the build instructions for wxPython if you are building wxGTK yourself for some recommendations for how to configure it.

    • In order to use the wxGLCanvas you'll need to have either OpenGL or the Mesa3D library on your system. Please note that the binary RPMs below have a dependency on the GL and GLU libraries so you'll need to have them on your system or the wxPython extension won't load. You can get the RPM for Mesa that I am using on my system from here. wxPython's wxGLCanvas only provides the GL Context and a wxWindow to put it in, so you will also need the PyOpenGL Python extension module as well.

      If you are building wxPython yourself and don't care to use OpenGL/Mesa then you can easily skip building it and can ignore this step. See the build instructions for details.

    Prebuilt Binaries
    Win32
    The Win32 version of wxPython is distributed as a standard self-installing executable. This allows you the convenience of using the Start Menu and Add/Remove Programs just like any other Windows software. Each installer contains the following:
    • The wxPython extension module and associated shadow class files
    • The wxWindows/wxPython documentation formatted as a MS HTMLHelp file
    • The wxPython library
    • The wxPython demo
    There are currently two versions available:
    • wxPython-2.2.2-Py15.EXE -- for Python 1.5.2
    • wxPython-2.2.2-Py20.EXE -- for Python 2.0

    Linux RPMs
    These files are for RedHat Linux, or other Linux systems that are able to use RPMs. They were built on a RedHat 6.2 system, your mileage may vary on other systems... These RPMs contain only the wxPython extension module and associated shadow class files, and the wxPython library. See below to get the demo and documentation.
    • wxPython-2.2.2-1-Py15.i386.rpm -- for Python 1.5.2, installs to /usr/lib/python1.5/site-packages/
    • wxPython-2.2.2-1-Py20.i386.rpm -- for Python 2.0, installs to /usr/local/lib/python2.0/site-packages/

    Debian
    Debian users can get wxPython from the Debian pacakge distribution system. It can be found here, or you can just use apt-get.

    Documentation
    This is the wxWindows/wxPython documentation, in HTML format. For other documentaion options see the wxWindows site.

    Demo
    Since the RPMs no longer include the demo, it is now packaged separately. Why did I do this? Well the demo is intended to be a learning tool, but when installed from the RPM all the demo files are owned by root. As a separate tarball you can put the files where ever you want and since they'll be owned by you when you untar them you can play and tweak to your heart's desire.
    • wxPython-demo-2.2.2.tar.gz

    Development Sources
    You can also get access to the source tree from the CVS server at SourceForge. This lets you have instant access to new features and bug fixes as I complete their implementaion. To learn more about CVS and to get the CVS programs, go to the CVS Home.

    To get check out your copy of the source code, simply run these commands, (assumes the command-line version of CVS, if you are using WinCVS or other GUI version, adapt accordingly.) When prompted for a password, just press ENTER. 

        cvs -d:pserver:anonymous@cvs.wxWindows.sourceforge.net:/cvsroot/wxwindows login 

    cvs -z3 -d:pserver:anonymous@cvs.wxWindows.sourceforge.net:/cvsroot/wxwindows co wxWindows

    You will find the wxPython sources in the wxWindows/wxPython directory. When you want to update your copy of the sources, simply change to the wxWindows directory and run 

        cvs update
