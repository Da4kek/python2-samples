Overview of JPython Documentation

There is a variety of documentation available to help you learn to use JPython. We will be expanding this information in the future so check back here often. You can use the sidebar you see on the left side of this page as a table of contents to easily jump to the documentation you're interested in. A more detailed description of these links is provided below.

JPython Specific Documentation

The following documentation is specific to JPython:
  • For a one-page introduction to what JPython is, and why you should care, check out What is JPython?.
  • Here is information on invoking the JPython interpreter by typing jpython at the shell prompt.
  • The JPython registry replaces environment variables for PYTHONPATH and more.
  • The embedding document describes how to call JPython from a Java application.
  • You might also be interested in building JPython from the source code.

Interaction with Java Packages

Most JPython applications will want to use the vast array of Java packages available. The following documentation helps you work with Java packages.
  • The Basics.
  • Working with JavaBean properties, making all Java classes easier to use from Python.
  • Special care is necessary to build and use Java arrays from Python.
  • This document describes how to subclass Java classes in Python.
  • The jpythonc utility compiles Python source code to real Java classes, and is used for building applets, servlets, beans, etc.

General Python Documentation

While JPython has only been publically available for eight months, the Python language has been around for many years. The majority of the standard Python documentation applies to JPython. A few very handy places to look are:

  • The Python Tutorial (start here)
  • The Python Library Reference. Although many of these modules are not yet implemented (those coded in C for CPython must be re-implemented in Java for JPython), experience Python programmers keep the Library Reference Manual under their pillow for easy access.

Other Useful Links

  • JPython and CPython are two different implementations of the same language. There are naturally some differences between the two implementations, sometimes reflecting undefined aspects of the Python language, sometimes reflecting quirks of the independent implementations. Here is a more detailed discussion and comparision of JPython and CPython.
  • The JPython FAQ may already contain the answer to your question...
  • ...and if it doesn't, then the JPython-interest mailing list archives might help.
  • You might also be interested in reading Jim Hugunin's JPython paper, which appeared in the proceedings of the 6th International Python Conference.

Peer-to-Peer Help

If you still have unanswered questions you can usually get help from:
  • The jpython-interest mailing list.
  • The comp.lang.python newsgroup.
  • JPython's developers, who can be reached at jpython@python.org.
